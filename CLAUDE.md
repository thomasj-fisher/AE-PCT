# AE-PCT engineering principles

Project: a static HTML + Leaflet PWA for tracking Athena's PCT 2026 thru-hike. Deployed via GitHub Pages from `main`. Every commit to `main` rebuilds the live URL — treat `main` as production.

## 1. Git workflow (load-bearing)

- **Branch per feature or fix.** `feat/<short-desc>`, `fix/<short-desc>`, `docs/<short-desc>`, `chore/<short-desc>`. Never commit non-trivial work directly to `main`.
- **PRs only.** Open a pull request with `gh pr create` and **wait for Thomas to merge it.** Don't run `gh pr merge` on his behalf. He may explicitly say "merge it" — only then.
- **Exception — single-line edits to `data/progress.js`.** When Thomas reports an Athena trail update and says "push it", a direct commit to `main` is acceptable. Anything else gets a branch.
- **Green commits only.** If you broke something while editing, fix it before committing — don't push the broken intermediate state.
- **Never skip hooks** (`--no-verify`) or bypass signing unless explicitly authorised.
- **Never amend pushed commits or force-push to `main`.**

## 2. Tidy First at phase boundaries (Kent Beck)

Before opening a PR:
1. End-to-end smoke check — load the page locally, click through the changed flow.
2. Remove dead code, comments, and console logs that were debugging scaffolding.
3. If behaviour changed, update this file or the affected data-file header comment.
4. Bump the service worker `CACHE` constant in `sw.js` if any shell file (HTML, CSS, JS, manifest, icon, or `data/*.js` other than progress) changed. This is the only way Athena's cached PWA picks up the change reliably.
5. Commit. Clean commits mark phase boundaries and make rollback trivial.

## 3. File header convention

Top of every JS / PS1 file:

```js
// FILE: <filename>
// PURPOSE: <one-sentence intent>
// SOURCE: <if data is derived from somewhere, cite it>
// CAVEATS: <known limitations>
```

Existing files (`app.js`, `data/water-loader.js`, `data/water-waypoints.js`, the build scripts) follow this pattern. Match it for new files.

## 4. Source citations on user-facing data

Any popup or table cell that surfaces information from an external source MUST link back to that source. The `sources: [...]` pattern on each `PCT_WAYPOINTS` entry is the contract — preserve it on every new field.

Current sources in use:
- Halfmile's PCT Resupply Notes (Nov 2023 PDF) — resupply detail
- pctwater.com Google Sheets — water reports
- Halfmile's PCT Maps GPX zips — water-source coordinates
- Yogi's *Hiking the Pacific Crest Trail* (130-day schedule) — daily mileage plan

Do not silently embed third-party content without citation. Paraphrase rather than copy verbatim. Never commit the copyrighted source material itself to the repo — paraphrased fields only.

## 5. Service worker discipline

- Strategy is **network-first** for same-origin requests, cache-fallback offline. Map tiles bypass the SW entirely.
- When the app shell changes, bump `CACHE = 'pct-tracker-vN'` in `sw.js`. Old caches get pruned on activate.
- Network-first means most users see the new version on next reload when online. **iOS Safari may need two reloads** to swap the active SW.

## 6. Tests

There are no tests yet. The site is small and visually verifiable. If we add tests:

- A `tests/` directory next to source.
- Unit tests use small hand-crafted fixtures, never real Athena progress data.
- Red → green → refactor. Never commit on red.
- Never write implementation code that is not driven by a test (once the test discipline is in place).

Until then, every code-bearing PR includes a manual `## Test plan` checklist in the PR body — the things Thomas should poke before merging.

## 7. Data files

- `data/trail-data.js` — 39,856 PCT coordinates, regenerated from `gpx/` by `scripts/build-trail-data.ps1`. Don't hand-edit.
- `data/waypoints-data.js` — 41 resupply / road-access waypoints, hand-curated. Each entry has a `sources` array.
- `data/water-waypoints.js` — 8,473 Halfmile waypoint coords, regenerated by `scripts/build-water-waypoints.ps1`. Don't hand-edit.
- `data/water-loader.js` — runtime fetch of pctwater.com Google Sheets.
- `data/progress.js` — Athena's actual trail progress. Hand-edited by Claude when Thomas relays an update.

When updating `progress.js`, sort entries by date. The most recent entry drives the "last reported" marker on the map.

## 8. Things to never do

- Commit secrets or credentials.
- Commit anything from `.claude/` (already gitignored).
- Commit the agent-scratch dumps of copyrighted source PDFs (already gitignored).
- Auto-merge PRs on Thomas's behalf — open them and wait.
- Update `git config` from inside a session.
